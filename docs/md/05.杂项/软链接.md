| 对比项                   | Windows                 | macOS / Linux                        |
| ------------------------ | ----------------------- | ------------------------------------ |
| **创建命令**             | `mklink link target`    | `ln -s target link`                  |
| **创建目录链接**         | `mklink /D link target` | `ln -s target link`（文件/目录通用） |
| **参数顺序**             | 链接在前，目标在后      | 目标在前，链接在后                   |
| **路径默认类型**         | 绝对路径                | 相对路径（建议用相对路径）           |
| **跨文件系统**           | ✅                       | ✅                                    |
| **需要管理员权限**       | 默认需要                | 不需要                               |
| **查看是否链接**         | `dir`（显示 ``）        | `ls -la`（显示 `->`）                |
| **删除链接**             | `del link` / `rm link`  | `rm link`                            |
| **删除不影响原文件**     | ✅                       | ✅                                    |
| **悬空链接（目标已删）** | 链接还在，访问报错      | 链接还在，`ls` 显示红色              |



## 注意：参数顺序是反的！

这是最容易踩的坑：

```bash
# Windows：链接名在前
mklink python.exe python3.12.exe
#        ↑链接     ↑目标

# macOS/Linux：目标在前，链接名在后
ln -s python3.12.exe python.exe
#        ↑目标             ↑链接
```
