## **一、举例说明如何理解 Skills 和 MCP**

你走进一家五金店，想要修理一个坏掉的橱柜。店里什么都有（木工胶、夹子、替换替换铰链），但是知道该买哪些物品以及如何使用它们却是另外一个问题。

MCP就像拥有进入货架通道的权限。与此同时，Skills则如同员工的专业技能。如果你不知道需要哪些物品或如何使用它们，即使仓库再多也无济于事。skill就像一位乐于助人的员工，它会指导你完成修理流程，指出合适的耗材，并且教你正确的操作方法。

更具体的说，MCP server赋予Claude访问外部系统、服务和平台的权限，而skills则为Claude提供有效利用这些连接所需的上下文信息，指导Claude在获得访问权限后应该做什么。

## **二、为什么 Skills 和 MCP 能很好地协同工作**

MCP负责连接：提供对外部系统的安全、标准化的访问。无论你连接的是GitHub、Salesforce、Notion，还是你自己的内部API，MCP server都能让Claude访问你的工具和数据。

Skills负责处理专业知识：将原生工具访问转化为可靠结果所需的领域知识和工作流逻辑。skill知道何时查询CRM、在结果中查找什么、如何格式化输出以及哪些特殊情况需要特殊处理。

这种分离保证了结构的可组合性。单个skill可以协调多个MCP server，而单个MCP server可以支持数十个不同的skills。添加新的连接后，现有的skill可以将其集成。改进skill后，它将适用于所有已连接的工具。

### **2.1 将 Skills和 MCP 结合起来，您将获得：**

**清晰的发现**：Claude不再猜测该从何处着手。一项会议准备skill可能会明确指出：首先查看项目页面，然后查看之前的会议记录，最后查看利益相关者资料。一项研究skill可能会指出：从共享驱动器开始，与客户关系管理系统 (CRM) 进行交叉比对，然后通过网络搜索填补空白。这项技能体现了机构知识，即哪些资源对哪些任务至关重要。

**可靠的流程编排**：多步骤工作流程变得可预测。如果没有相应的skill，Claude 可能会在检查数据是否完整之前就提取并格式化数据。skills会明确定义流程顺序，因此 Claude 每次都能以相同的方式执行工作流程。

**持续稳定的表现**：产出真正符合标准。千篇一律的结果需要修改。skills决定了团队对“完成”的定义：合适的结构、恰当的细节程度以及适合受众的语气。

随着时间的推移，团队会积累各种相互关联的skills和人脉，使Claude在其特定领域拥有专业知识。

### **2.2 Skills 和 MCP 可能重叠的**

MCP 服务器可能包含工具使用提示和常用任务指导等形式的说明。这样可以将特定于工具的知识与工具紧密结合。但是，这些说明的设计初衷应该是通用的。

一般来说：MCP 指南涵盖如何正确使用服务器及其工具。skill 指南则涵盖如何将它们用于特定流程或在多服务器工作流中使用。

例如，Salesforce MCP 服务器可能会指定查询语法和 API 格式。skill 则会指定首先检查哪些记录、如何将它们与 Slack 对话交叉引用以获取最新上下文，以及如何组织输出以供团队进行流程审查。

将 MCP server 和 skills 结合使用时，请注意是否存在冲突的指令。如果 MCP server指示返回 JSON 格式，而skill指示将其格式化为 Markdown 表格，那么 Claude 就必须猜测哪个才是正确的。让 MCP 处理连接，让 skills 处理呈现、排序和工作流逻辑。

**延伸阅读：**了解 skills 如何使用渐进式披露按需加载上下文以及程序化工具调用来高效地协调 MCP 工具。

## **三、将 Skills 和 MCP 结合使用的实际案例**

现在我们来看看 skills 和 MCP 如何在实际工作流程中结合使用。我们将通过两个例子来讲解：财务分析师提取实时市场数据进行公司估值，以及项目经理使用 Notion 的会议智能skill进行会议准备。

在这两种情况下，MCP server提供对工具的访问权限，而skills则定义了如何使用这些工具。

### **3.1 财务分析：公司估值自动化Skill**

Anthropic 发布了一套预置技能(pre-build skills)，用于常见的财务工作流程，包括可比公司分析。可比公司分析是一种标准的估值方法。分析师在进行可比公司分析时，需要花费数小时从多个来源提取财务指标，应用相同的估值方法，并格式化输出以符合合规标准。这项工作重复性高、容易出错，而这正是 skills 和 MCP 协同工作能够显著提升效率的工作流程。

**Skill**：可比公司分析可自动执行此估值工作流程，从多个来源提取数据，应用一致的方法，并将输出格式化为特定标准。

**MCP servers**：连接到S&P Capital IQ、Daloopa和Morningstar以获取实时市场数据

**工作流程**：

- Skill 用于确定要查询哪些数据源（发现）
- MCP 连接获取实时财务数据
- Skill 应用方法论并格式化输出（编排）
- Skill 符合合规要求（绩效）

### **3.2 会议准备：Notion 的会议智能 Skill**

会议准备工作很繁琐。你需要从多个渠道收集信息，例如项目文档、之前的会议记录以及利益相关者的信息，然后将其整合到会前阅读材料和议程中。这是一个多步骤的过程，每次你都得重新解释一遍。

**Skill：**会议智能定义了要搜索的页面、输出的结构以及要包含的章节。

**MCP server**：用于搜索、读取和创建页面的 Notion 连接

**工作流程**：

- 该 skill 可识别要搜索的相关页面，包括项目、以往会议、利益相关者信息（发现）
- MCP 连接从 Notion 搜索并检索内容
- Skill 结构包含两份文件：内部预读文件和外部议程（统筹安排）。
- MCP 连接会将两个文档保存到 Notion 中，并进行整理和链接。
- Skill 确保输出符合格式标准（性能）

## **四、何时使用 Skills 而非 MCP**

Skills 和 MCP 解决的是不同的问题，但对于特定的工作流程来说，决定使用哪一个并不总是显而易见的。

### **4.1 Skills 的用途是什么？**

Skills 是指那些原本只存在于你脑海中，或者每次有新成员加入团队都需要重新解释的知识。它们最适合用于：

- 涉及工具的多步骤工作流程：会议准备工作从多个来源提取信息，然后创建结构化文档
- 需要保持一致性的流程：季度财务分析必须每次都遵循相同的方法，合规性审查设有强制性检查点。
- 您希望收集和分享的领域专业知识：研究方法、代码审查标准、写作指南
- 即使团队成员离职，也应该保留的工作流程：编码在可重用指令中的机构知识

### **4.2 MCP server的用途**

MCP 扩展了 Claude 的访问权限和使用权限。当您需要以下情况时，请使用 MCP：

- 实时数据访问：搜索 Notion 页面、阅读 Slack 消息、查询数据库
- 外部系统中的操作：创建 GitHub 问题、更新项目管理工具、发送通知
- 文件操作：从 Google 云端硬盘读取数据和写入数据，访问本地文件系统
- API 集成：连接到不具备原生 Claude 支持的服务

如果你是在解释如何做某件事，那是一种 skill。如果你需要Claude访问某个东西，那就是多级协作（MCP）。

### **4.3 快速参考表：Skills 和 MCP 的区别**

|            | Skills                 | MCP                          |
| ---------- | ---------------------- | ---------------------------- |
| 它是什么   | 程序性知识             | 工具连接                     |
| 它的作用   | 教Claude如何做某事     | 让Claude有机会接触到某些东西 |
| 何时加载   | 按需，在相关情况下     | 连接后始终可用               |
| 包含内容   | 说明、脚本、模板、资源 | 工具、资源、提示             |
| Token 行为 | 按需加载，保留上下文   | 预先加载定义                 |
| 最适合场景 | 工作流程、标准、方法论 | 数据访问、API 调用、外部操作 |

## **五、常见问题解答**

### **5.1 Skills 可以取代 MCP 吗？**

不。skills 和 MCP 解决的是不同的问题。MCP 提供与外部工具和数据的连接。skills 提供如何有效使用这种连接的操作知识。大多数强大的工作流程都会同时使用两者。

### **5.2 一个 skill 可以使用多个MCP server吗？**

是的。一项 skill 可以同时协调多个MCP server。例如，一项技术竞争分析 skill 可能需要搜索Google Drive以获取内部研究资料，从GitHub拉取竞争对手的代码库，并通过网络搜索收集市场数据。

### **5.3 我可以为一个MCP server创建多个 skills吗？**

是的。一项skill可以提升您从单个 MCP 连接中获得的价值。Notion 通过会议准备、研究、知识获取和规范到实施等独立技能展示了这种模式——点击此处查看。
