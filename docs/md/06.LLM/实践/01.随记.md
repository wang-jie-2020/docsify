### 配置

- GitHub 上找到1-2个配置指南

- 全局范围(SKILLS、COMMAND、RULES、AGENTS)
  - superpower-skills

    ├─brainstorming --- 创建功能/组件前的需求探索与设计

    ├─dispatching-parallel-agents --- 并行执行多个独立任务

    ├─executing-plans --- 在独立 session 中按计划执行实现，带检查点

    ├─finishing-a-development-branch --- 实现完成后，指导合并/PR/清理

    ├─receiving-code-review --- 处理 code review 反馈，验证后实施

    ├─requesting-code-review --- 完成任务或合并前请求 code review

    ├─subagent-driven-development --- 在 session 内执行含独立任务的实现计划

    ├─systematic-debugging --- 遇到 bug 或测试失败时，系统性排查

    ├─test-driven-development --- 实现前先写测试

    ├─using-git-worktrees --- 为 feature 工作创建隔离的 git worktree

    ├─using-superpowers --- 每次会话开始时的引导 skill

    ├─verification-before-completion --- 声称完成前必须跑验证，证据先于断言

    ├─writing-plans --- 有 spec 后、写代码前，制定实现计划

    └─writing-skills --- 创建/编辑/验证 skills
