# JSON

## FastJson2

```xml
<dependency>
    <groupId>com.alibaba.fastjson2</groupId>
    <artifactId>fastjson2</artifactId>
</dependency>
```

*毁誉参半, 在api语法设计上明显会简便很多, 当然有方法可以配置为mvc的messageConverter, 必要性么就不再叙述了*



核心API:

| 方法                                                     | 说明                              |
| -------------------------------------------------------- | --------------------------------- |
| `JSON.toJSONString(obj)`                                 | 对象 → JSON字符串                 |
| `JSON.parseObject(json, Class)`                          | JSON字符串 → 对象（指定类型）     |
| `JSON.parseArray(json, Class)`                           | JSON字符串 → List（指定元素类型） |
| `JSON.parseObject(json, new TypeReference<List<T>>(){})` | 泛型集合反序列化                  |
| `JSONPath.read(json, "$.user.name")`                     | JSONPath路径查询                  |
| `JSONB.toBytes(obj)`                                     | 序列化为JSONB二进制（高性能场景） |
| `JSONB.parseObject(bytes, Class)`                        | 从JSONB二进制解析                 |



常见注解:

@JSONField(name="resType")										字段名映射

@JSONField(format="yyyy-MM-dd")								格式化, 通常处理日期

@JSONField(serialize = false)									不参与序列化

@JSONField(deserialize = false)								不参与反序列化

@JSONField(alternateNames = {"name", "userName"})				反序列化时接受多个别名



常见使用场景:

(1) 基础序列化/反序列化 — API响应、请求体解析、Redis缓存、日志记录

```java
// 序列化
String json = JSON.toJSONString(user);

// 反序列化
User user = JSON.parseObject(json, User.class);

// 泛型集合 — 必须用 TypeReference，否则泛型擦除后返回 LinkedHashMap
List<User> list = JSON.parseObject(json, new TypeReference<List<User>>(){});
Map<String, List<User>> map = JSON.parseObject(json, new TypeReference<Map<String, List<User>>>(){});
```

(2) 动态JSON操作 — 解析不确定结构的第三方数据

```java
JSONObject obj = JSON.parseObject(jsonStr);
String name = obj.getString("name");
Integer age = obj.getInteger("age");
JSONArray arr = obj.getJSONArray("items");
```

(3) JSONB二进制格式 — 高性能场景（Redis缓存、RPC传输、消息队列）

> 体积比JSON字符串小30%~50%, 序列化/反序列化速度快3~5倍

```java
byte[] bytes = JSONB.toBytes(user);                    // 序列化
User user = JSONB.parseObject(bytes, User.class);      // 反序列化
```

(4) JSONPath查询 — 从大JSON中提取部分字段, 无需完整反序列化

```java
// 提取嵌套字段
String name = (String) JSONPath.eval(json, "$.data.user.name");

// 提取数组中满足条件的元素
List<Object> result = JSONPath.eval(json, "$.data.list[?(@.status == 1)]");
```

(5) Feature特性控制 — 灵活控制序列化/反序列化行为

```java
// 序列化时控制
JSON.toJSONString(obj,
    JSONWriter.Feature.PrettyFormat,          // 格式化输出（仅开发环境使用）
    JSONWriter.Feature.WriteMapNullValue,     // 输出null字段
    JSONWriter.Feature.WriteNullStringAsEmpty // null字符串输出为""
);

// 反序列化时控制
JSON.parseObject(json, User.class,
    JSONReader.Feature.IgnoreCaseMatch,       // 字段名大小写不敏感
    JSONReader.Feature.IgnoreNotMatch         // 忽略不匹配的字段
);
```

(6) 全局配置

```java
JSON.configWriterDateFormat("yyyy-MM-dd HH:mm:ss");
JSON.register(LocalDateTime.class, LocalDateTimeReader.INSTANCE);
JSONFactory.setUseJacksonAnnotation(false);     // 禁用Jackson注解支持
```

在函数签名中可以传递更多配置:

```java
static String toJSONString(Object object, Filter[] filters, JSONWriter.Feature... features)
```



常见错误:

(1) 泛型擦除 — 最常见的错误

```java
// ❌ 错误: 泛型擦除, 返回 List<LinkedHashMap> 而不是 List<User>
List<User> list = JSON.parseArray(json, User.class);

// ✅ 正确: 用 TypeReference 保留泛型信息
List<User> list = JSON.parseObject(json, new TypeReference<List<User>>(){});
```

(2) 安全性 — autoType反序列化漏洞

> FastJson1有严重的反序列化漏洞(CVE-2020-8840等), FastJson2已默认禁用autoType

```java
// ❌ 危险: 无类型约束, 存在反序列化漏洞风险
JSON.parse(json);

// ✅ 安全: 必须指定目标类型
JSON.parseObject(json, User.class);
```

*永远不要使用 JSON.parse(), 始终指定目标类*

(3) 缺少无参构造器

```java
// ❌ 报错: default constructor not found
public class User {
    private String name;
    public User(String name) { this.name = name; } // 只有有参构造
}

// ✅ 正确: 必须有无参构造器 + getter/setter
public class User {
    private String name;
    public User() {}   // 必须有无参构造
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
}
```

(4) 日期格式

> 默认序列化为Long时间戳, 不是可读日期字符串

```java
@JSONField(format = "yyyy-MM-dd HH:mm:ss")
private Date createTime;
```

*与Jackson不同, FastJson2对日期格式要求严格, 如 "2025-11-28 09:35:57.257" 带毫秒的字符串可能解析失败*

(5) 字段名大小写不匹配

```java
// JSON: {"Name":"张三", "AGE":28}  →  Java: name, age
// ❌ 默认大小写敏感, 无法映射

// ✅ 方案1: 注解
@JSONField(name = "Name")
private String name;

// ✅ 方案2: 反序列化Feature
JSON.parseObject(json, User.class, JSONReader.Feature.IgnoreCaseMatch);
```

(6) JSONB字段不匹配

> JSONB反序列化要求字段严格匹配类结构, 多余或缺失字段会报错

```java
// ❌ JSONB中有多余字段时可能报错
User user = JSONB.parseObject(bytes, User.class);

// ✅ 使用 Feature.IgnoreNotMatch 容忍
JSONB.parseObject(bytes, User.class, JSONReader.Feature.IgnoreNotMatch);
```

(7) SpringBoot中与Jackson注解冲突

> 默认情况下FastJson2会识别Jackson注解(如@JsonProperty), 可能导致预期外的行为

```java
// 禁用Jackson注解支持
JSONFactory.setUseJacksonAnnotation(false);
```

(8) 特殊字符解析异常

> 包含特殊字符(如换行符、未转义引号)的JSON字符串可能报 syntax error

```java
// 确保JSON字符串合法, 或在写入前做好转义
// 对于不可控的第三方数据, 建议做 try-catch 处理
try {
    return JSON.parseObject(json, Target.class);
} catch (JSONException e) {
    // 记录原始数据, 降级处理
    log.error("JSON解析失败: {}", json, e);
}
```
