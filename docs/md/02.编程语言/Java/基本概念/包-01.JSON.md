# JSON

## Jackson

```xml
<!-- **jackson-databind** 包含了 **jackson-annotations** **jackson-core** -->
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
</dependency>
```

*默认包含在web包中, 可以不再考虑单独新增, 但必须指出的在非SpringBoot场景下需要添加jsr310(比如LocalDateTime类型)同时进行配置处理*



常见的注解标记:

@JsonProperty("xxx")								字段不一致

@JsonValue										  字段值, 通常会考虑在枚举中替换默认输出

@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")		格式化, shape + pattern, 通常处理枚举和日期

@JsonSerialize(using = ToStringSerializer.class)	控制序列化, 相对应的还有@JsonDeserialize

@JsonIgnore  @JsonIgnoreProperties  @JsonIgnoreType 控制忽略

@JsonInclude(JsonInclude.Include.NON_NULL)			为 null 时不序列化输出该字段

@JsonAlias({"name", "userName"})					反序列化时接受多个别名字段名

@JsonManagedReference @JsonBackReference			控制循环引用

@JacksonAnnotationsInside  						 内联标记 --> 标记了@Sensitive 就像直接在那个元素上使用了 @JsonSerialize



常见的配置场景:

(1) 时间

>在默认情况下:
>
>​	Date			-> 2025-11-28T08:06:32.915+00:00
>
>​	LocalDateTime   -> 2025-11-28T16:06:32.9155581
>
>
>
>​	Date			<- 2025-11-28T08:06:32.915+00:00
>
>​	Date			<- 2025-11-28T16:06:32.9155581
>
>​	Date			<- 2025-11-28T16:06:32.915Z
>
>
>
>​	LocalDateTime   <- 2025-11-28T16:06:32.9155581
>
>​	LocalDateTime   <- 2025-11-28T16:06:32.915Z

spring.jackson.date-format = yyyy-MM-dd HH:mm:ss 

builder.simpleDateFormat("yyyy-MM-dd HH:mm:ss") 

​	影响Date(序列化和反序列化), 但是不影响LocalDateTime

@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")

​	影响Date和LocalDateTime(序列化和反序列化),

​	Date的转换宽松一些,比如 "2025-11-28 09:35:57.257" Date可以接收但是LocalDateTime不可以接收



存在全局的控制如下:

```java
@Bean
public Jackson2ObjectMapperBuilderCustomizer jackson2ObjectMapperBuilderCustomizer() {
    return builder -> {
        builder.simpleDateFormat("yyyy-MM-dd HH:mm:ss");
        builder.serializerByType(LocalDateTime.class, 
                                 new LocalDateTimeSerializer(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")));
        builder.deserializerByType(LocalDateTime.class, 
                                   new LocalDateTimeDeserializer(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")));
    };
}
```

(3) LONG

```java

builder.serializerByType(Long.class, new LongSerializer()); // extends JsonSerializer<Long> 
```

也可以用内置的 ToStringSerializer 直接注册，不需要自定义:

```java

builder.serializerByType(Long.TYPE, ToStringSerializer.instance);
```

(4) 配置重写

**两种方式的区别：**

| 对比 | `Jackson2ObjectMapperBuilderCustomizer` | `MappingJackson2HttpMessageConverter` |
|------|----------------------------------------|--------------------------------------|
| 作用层级 | 定制 ObjectMapper 的构建过程 | 定制 HTTP 消息转换器本身 |
| 影响范围 | 全局（MVC + 手动注入的 ObjectMapper 均一致） | 仅影响 Spring MVC HTTP 请求/响应 |
| 侵入性 | 低，在原有基础上叠加 | 高，通常需要完全替换转换器 |
| 多个叠加 | 支持多个 Bean，按 @Order 顺序叠加 | 不适合叠加 |
| 推荐程度 | ✅ 首选 | ⚠️ 特殊场景才用 |

**创建链路：**

```
Jackson2ObjectMapperBuilderCustomizer（定制 Builder）
        ↓
Jackson2ObjectMapperBuilder（build 出）
        ↓
ObjectMapper
        ↓
MappingJackson2HttpMessageConverter
        ↓
Spring MVC HTTP 请求/响应 JSON 转换
```

Customizer 在上游影响"原材料"，Converter 在下游处理 HTTP；用 Customizer 配置后，MVC 和手动注入的 ObjectMapper 行为一致，用 Converter 新建 ObjectMapper 则两者会不一致。

`MappingJackson2HttpMessageConverter` 真正需要的场景：同一应用需要两套序列化规则（不同接口不同策略）、需要支持特殊 MediaType、对接遗留系统需要 HTTP 层特殊处理但不影响内部逻辑。

(5) 自定义序列化器

```java
// 示例：金额以分存储，序列化时输出为元
public class CentToYuanSerializer extends JsonSerializer<Long> {
    @Override
    public void serialize(Long value, JsonGenerator gen,
                          SerializerProvider provider) throws IOException {
        gen.writeString(new BigDecimal(value)
            .divide(new BigDecimal(100))
            .setScale(2, RoundingMode.HALF_UP)
            .toString());
    }
}

// 使用
public class Order {
    @JsonSerialize(using = CentToYuanSerializer.class)
    private Long amount; // 存储: 10000 → 序列化: "100.00"
}
```

(6) 手动序列化 / 反序列化

```java
@Autowired
ObjectMapper objectMapper;

// 对象 → JSON 字符串
String json = objectMapper.writeValueAsString(obj);

// JSON 字符串 → 对象
User user = objectMapper.readValue(json, User.class);

// JSON 字符串 → List
List<User> list = objectMapper.readValue(json,
    objectMapper.getTypeFactory().constructCollectionType(List.class, User.class));

// JSON 字符串 → Map
Map<String, Object> map = objectMapper.readValue(json, new TypeReference<Map<String, Object>>() {});
```

*直接注入 ObjectMapper Bean 使用，而不是 new ObjectMapper()，保证和全局配置一致*

## FastJson2

```xml
<dependency>
    <groupId>com.alibaba.fastjson2</groupId>
    <artifactId>fastjson2</artifactId>
</dependency>
```

*毁誉参半, 在api语法设计上明显会简便很多, 当然有方法可以配置为mvc的messageConverter, 必要性么就不再叙述了*



常见注解:

@JSONField(name="resType")										字段名映射

@JSONField(format="yyyy-MM-dd")								格式化, 通常处理日期

@JSONField(serialize = false)									不参与序列化

@JSONField(deserialize = false)								不参与反序列化

@JSONField(alternateNames = {"name", "userName"})				反序列化时接受多个别名



核心API:

| 方法 | 说明 |
|------|------|
| `JSON.toJSONString(obj)` | 对象 → JSON字符串 |
| `JSON.parseObject(json, Class)` | JSON字符串 → 对象（指定类型） |
| `JSON.parseArray(json, Class)` | JSON字符串 → List（指定元素类型） |
| `JSON.parseObject(json, new TypeReference<List<T>>(){})` | 泛型集合反序列化 |
| `JSONPath.read(json, "$.user.name")` | JSONPath路径查询 |
| `JSONB.toBytes(obj)` | 序列化为JSONB二进制（高性能场景） |
| `JSONB.parseObject(bytes, Class)` | 从JSONB二进制解析 |



常见使用场景:

(1) 基础序列化/反序列化 — API响应、请求体解析、Redis缓存、日志记录

```java
// 序列化
String json = JSON.toJSONString(user);

// 反序列化
User user = JSON.parseObject(json, User.class);

// 泛型集合 — 必须用 TypeReference，否则泛型擦除后返回 LinkedHashMap
List<User> list = JSON.parseObject(json, new TypeReference<List<User>>(){});
Map<String, List<User>> map = JSON.parseObject(json, new TypeReference<Map<String, List<User>>>(){});
```

(2) 动态JSON操作 — 解析不确定结构的第三方数据

```java
JSONObject obj = JSON.parseObject(jsonStr);
String name = obj.getString("name");
Integer age = obj.getInteger("age");
JSONArray arr = obj.getJSONArray("items");
```

(3) JSONB二进制格式 — 高性能场景（Redis缓存、RPC传输、消息队列）

> 体积比JSON字符串小30%~50%, 序列化/反序列化速度快3~5倍

```java
byte[] bytes = JSONB.toBytes(user);                    // 序列化
User user = JSONB.parseObject(bytes, User.class);      // 反序列化
```

(4) JSONPath查询 — 从大JSON中提取部分字段, 无需完整反序列化

```java
// 提取嵌套字段
String name = (String) JSONPath.eval(json, "$.data.user.name");

// 提取数组中满足条件的元素
List<Object> result = JSONPath.eval(json, "$.data.list[?(@.status == 1)]");
```

(5) Feature特性控制 — 灵活控制序列化/反序列化行为

```java
// 序列化时控制
JSON.toJSONString(obj,
    JSONWriter.Feature.PrettyFormat,          // 格式化输出（仅开发环境使用）
    JSONWriter.Feature.WriteMapNullValue,     // 输出null字段
    JSONWriter.Feature.WriteNullStringAsEmpty // null字符串输出为""
);

// 反序列化时控制
JSON.parseObject(json, User.class,
    JSONReader.Feature.IgnoreCaseMatch,       // 字段名大小写不敏感
    JSONReader.Feature.IgnoreNotMatch         // 忽略不匹配的字段
);
```

(6) 全局配置

```java
JSON.configWriterDateFormat("yyyy-MM-dd HH:mm:ss");
JSON.register(LocalDateTime.class, LocalDateTimeReader.INSTANCE);
JSONFactory.setUseJacksonAnnotation(false);     // 禁用Jackson注解支持
```

在函数签名中可以传递更多配置:

```java
static String toJSONString(Object object, Filter[] filters, JSONWriter.Feature... features)
```



常见错误:

(1) 泛型擦除 — 最常见的错误

```java
// ❌ 错误: 泛型擦除, 返回 List<LinkedHashMap> 而不是 List<User>
List<User> list = JSON.parseArray(json, User.class);

// ✅ 正确: 用 TypeReference 保留泛型信息
List<User> list = JSON.parseObject(json, new TypeReference<List<User>>(){});
```

(2) 安全性 — autoType反序列化漏洞

> FastJson1有严重的反序列化漏洞(CVE-2020-8840等), FastJson2已默认禁用autoType

```java
// ❌ 危险: 无类型约束, 存在反序列化漏洞风险
JSON.parse(json);

// ✅ 安全: 必须指定目标类型
JSON.parseObject(json, User.class);
```

*永远不要使用 JSON.parse(), 始终指定目标类*

(3) 缺少无参构造器

```java
// ❌ 报错: default constructor not found
public class User {
    private String name;
    public User(String name) { this.name = name; } // 只有有参构造
}

// ✅ 正确: 必须有无参构造器 + getter/setter
public class User {
    private String name;
    public User() {}   // 必须有无参构造
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
}
```

(4) 日期格式

> 默认序列化为Long时间戳, 不是可读日期字符串

```java
@JSONField(format = "yyyy-MM-dd HH:mm:ss")
private Date createTime;
```

*与Jackson不同, FastJson2对日期格式要求严格, 如 "2025-11-28 09:35:57.257" 带毫秒的字符串可能解析失败*

(5) 字段名大小写不匹配

```java
// JSON: {"Name":"张三", "AGE":28}  →  Java: name, age
// ❌ 默认大小写敏感, 无法映射

// ✅ 方案1: 注解
@JSONField(name = "Name")
private String name;

// ✅ 方案2: 反序列化Feature
JSON.parseObject(json, User.class, JSONReader.Feature.IgnoreCaseMatch);
```

(6) JSONB字段不匹配

> JSONB反序列化要求字段严格匹配类结构, 多余或缺失字段会报错

```java
// ❌ JSONB中有多余字段时可能报错
User user = JSONB.parseObject(bytes, User.class);

// ✅ 使用 Feature.IgnoreNotMatch 容忍
JSONB.parseObject(bytes, User.class, JSONReader.Feature.IgnoreNotMatch);
```

(7) SpringBoot中与Jackson注解冲突

> 默认情况下FastJson2会识别Jackson注解(如@JsonProperty), 可能导致预期外的行为

```java
// 禁用Jackson注解支持
JSONFactory.setUseJacksonAnnotation(false);
```

(8) 特殊字符解析异常

> 包含特殊字符(如换行符、未转义引号)的JSON字符串可能报 syntax error

```java
// 确保JSON字符串合法, 或在写入前做好转义
// 对于不可控的第三方数据, 建议做 try-catch 处理
try {
    return JSON.parseObject(json, Target.class);
} catch (JSONException e) {
    // 记录原始数据, 降级处理
    log.error("JSON解析失败: {}", json, e);
}
```
