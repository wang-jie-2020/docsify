# 对象映射（Object Mapping）

## 概述

对象映射 = 把一个Java对象的属性值复制到另一个Java对象中，典型场景如 Entity → DTO → VO 转换。


## 方案对比

| 方案 | 原理 | 性能 | 类型安全 | 编译检查 | 复杂映射 | 学习成本 |
|------|------|------|---------|---------|---------|---------|
| 手动 setter | 直接调用 | ⭐⭐⭐⭐⭐ | ✅ | ✅ | ✅ | 低 |
| MapStruct | 编译时代码生成 | ⭐⭐⭐⭐⭐ | ✅ | ✅ | ✅ | 中 |
| ModelMapper | 运行时反射+配置 | ⭐⭐⭐ | ❌ | ❌ | ✅ | 中 |
| Spring BeanUtils | 运行时反射 | ⭐⭐ | ❌ | ❌ | ❌ | 低 |
| Apache BeanUtils | 运行时反射 | ⭐ | ❌ | ❌ | ❌ | 低 |
| Hutool BeanUtil | 运行时反射 | ⭐⭐ | ❌ | ❌ | 部分 | 低 |

**结论：新项目核心模块用 MapStruct，需要复杂映射配置且非性能敏感场景可用 ModelMapper，简单场景可以用 BeanUtils，避免 Apache BeanUtils**


## BeanUtils.copyProperties

```java
// Spring: 源在左，目标在右
BeanUtils.copyProperties(source, target);

// Apache: 目标在左，源在右（注意参数顺序相反！）
org.apache.commons.beanutils.BeanUtils.copyProperties(target, source);
```

### 常见坑

(1) 参数顺序 — Spring 和 Apache 两个包的参数顺序**完全相反**

```java
// 如果项目同时引入了两个包，IDE自动导入可能导致静默复制失败
// 建议：团队统一使用一种，或用全限定类名调用
org.springframework.beans.BeanUtils.copyProperties(source, target);
```

(2) 类型不匹配 — 同名字段类型不同时直接跳过，不会报错也不会转换

```java
// source.age: Long    target.age: Integer
BeanUtils.copyProperties(source, target);
target.getAge(); // null，复制失败但无任何提示
```

(3) 浅拷贝 — 引用类型只复制引用，修改目标会影响源

```java
source.setAddress(new Address("北京"));
BeanUtils.copyProperties(source, target);

source.getAddress().setCity("上海");
target.getAddress().getCity(); // "上海" — 被污染了
```

(4) null值覆盖 — 源的null会覆盖目标的原有值

```java
source.setRemark(null);
target.setRemark("已有备注");
BeanUtils.copyProperties(source, target);
target.getRemark(); // null — 被覆盖了

// 解决：用 Hutool
BeanUtil.copyProperties(source, target, CopyOptions.create().ignoreNullValue());
```

(5) boolean + is前缀 — boolean基本类型的is前缀会被去掉，导致属性名不匹配

```java
// source: private boolean isActive  → setter是 setActive()
// target: private Boolean isActive  → setter是 setIsActive()
// 属性名不匹配，复制失败
```

*用包装类 Boolean 代替基本类型 boolean 可以避免此问题*

(6) 不同内部类 — 即使内部类名和属性完全相同，不同外部类的内部类也无法复制

```java
SourceA.Inner ≠ SourceB.Inner  // Spring认为是不同类型，赋值null
```

(7) 泛型属性 — 泛型类型不同时复制失败

```java
// source: List<SourceA.Inner>
// target: List<SourceB.Inner>
// 泛型不匹配，赋值null
```

(8) 性能 — 基于反射，性能远低于手写（差距可达100倍）

> 10万次操作：手动setter ~3ms，BeanUtils ~331ms
> 性能敏感场景（高并发、批量处理）应避免使用

(9) 静默失败 — 大多数失败场景不抛异常，只跳过该字段，很难排查

### 适用场景

- 字段简单的简单DTO转换
- 非性能敏感路径
- 快速原型/临时代码

**不适用场景：** 高并发、批量处理、复杂对象图、需要类型安全的场景


## MapStruct

### 基本使用

```xml
<dependency>
    <groupId>org.mapstruct</groupId>
    <artifactId>mapstruct</artifactId>
    <version>1.5.5.Final</version>
</dependency>
<!-- 编译时注解处理器 -->
<dependency>
    <groupId>org.mapstruct</groupId>
    <artifactId>mapstruct-processor</artifactId>
    <version>1.5.5.Final</version>
    <scope>provided</scope>
</dependency>
```

```java
@Mapper  // 编译时自动生成 UserMapperImpl 实现类
public interface UserMapper {
    UserMapper INSTANCE = Mappers.getMapper(UserMapper.class);

    UserDTO toDTO(UserEntity entity);
    UserEntity toEntity(UserDTO dto);
    List<UserDTO> toDTOList(List<UserEntity> entities); // 批量转换
}
```

### 常用功能

(1) 字段名不同

```java
@Mapping(source = "userName", target = "name")
@Mapping(source = "createTime", target = "gmtCreate")
UserDTO toDTO(UserEntity entity);
```

(2) 自定义类型转换

```java
@Mapper
public interface OrderMapper {
    @Mapping(target = "statusDesc", expression = "java(mapStatus(entity.getStatus()))")
    OrderDTO toDTO(OrderEntity entity);

    default String mapStatus(Integer status) {
        return status == 1 ? "启用" : "禁用";
    }
}
```

(3) 多源映射

```java
@Mapping(source = "user.name", target = "userName")
@Mapping(source = "dept.deptName", target = "deptName")
UserDetailDTO toDTO(UserEntity user, DeptEntity dept);
```

(4) 嵌套对象自动处理

```java
@Mapper(uses = {AddressMapper.class})  // 引用其他Mapper
public interface UserMapper {
    UserDTO toDTO(UserEntity entity);  // 内部会自动调用 AddressMapper
}
```

(5) 集合映射

```java
List<UserDTO> toDTOList(List<UserEntity> entities);  // 直接声明即可
Set<UserDTO> toDTOSet(Set<UserEntity> entities);
```

(6) 忽略字段

```java
@Mapping(target = "password", ignore = true)
@Mapping(target = "salt", ignore = true)
UserVO toVO(UserEntity entity);
```

(7) 默认值

```java
@Mapping(target = "status", defaultValue = "1")
UserDTO toDTO(UserEntity entity);
```

### Spring 集成

```java
@Mapper(componentModel = "spring")  // 生成 @Component，可直接注入
public interface UserMapper {
    UserDTO toDTO(UserEntity entity);
}

// 直接注入使用
@Service
public class UserService {
    @Autowired
    private UserMapper userMapper;
}
```

### 与 BeanUtils 对比优势

| 维度 | MapStruct | BeanUtils |
|------|-----------|-----------|
| 性能 | 接近手写（编译时生成代码） | 反射，慢100倍 |
| 类型安全 | 编译时检查，字段不匹配直接报错 | 运行时才失败，大多静默跳过 |
| 重构友好 | IDE自动联动 | 无法通过引用查找 |
| 嵌套对象 | 自动处理 | 浅拷贝，引用共享 |
| 可调试 | 看生成的Impl代码即可 | 反射调用栈难以跟踪 |

### 注意事项

- MapStruct 生成的代码在 `target/generated-sources/` 中，遇到问题先看生成的 Impl 类
- 编译时注解处理器需要正确配置（maven-compiler-plugin 或 gradle 的 annotationProcessor）
- Lombok 和 MapStruct 存在编译顺序问题，Lombok 插件需放在 MapStruct 前面

```xml
<!-- maven-compiler-plugin 中确保 Lombok 在 MapStruct 之前 -->
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <annotationProcessorPaths>
            <path>
                <groupId>org.projectlombok</groupId>
                <artifactId>lombok</artifactId>
                <version>${lombok.version}</version>
            </path>
            <path>
                <groupId>org.mapstruct</groupId>
                <artifactId>mapstruct-processor</artifactId>
                <version>${mapstruct.version}</version>
            </path>
        </annotationProcessorPaths>
    </configuration>
</plugin>
```


## Hutool BeanUtil

> Hutool 的 BeanUtil 是对反射拷贝的增强封装，解决了部分 BeanUtils 的痛点

```xml
<dependency>
    <groupId>cn.hutool</groupId>
    <artifactId>hutool-all</artifactId>
</dependency>
```

```java
// 基本用法（和 BeanUtils 类似，但支持更多特性）
BeanUtil.copyProperties(source, target);

// 忽略 null 值（解决 null 覆盖问题）
BeanUtil.copyProperties(source, target, CopyOptions.create().ignoreNullValue());

// 忽略特定字段
BeanUtil.copyProperties(source, target, CopyOptions.create().ignoreFields("password", "salt"));

// 仅复制指定字段
BeanUtil.copyProperties(source, target, CopyOptions.create().setFieldMapping(Map.of("oldName", "newName")));
```

*适合需要快速解决 null 覆盖问题但不想引入 MapStruct 的场景*


## ModelMapper

> 运行时反射 + 智能匹配，支持复杂映射配置，灵活性高但性能不如 MapStruct

### 基本使用

```xml
<dependency>
    <groupId>org.modelmapper</groupId>
    <artifactId>modelmapper</artifactId>
    <version>3.1.1</version>
</dependency>
```

```java
ModelMapper modelMapper = new ModelMapper();

// 同名字段自动映射
UserDTO dto = modelMapper.map(entity, UserDTO.class);
```

### Spring 集成

```java
@Bean
public ModelMapper modelMapper() {
    ModelMapper mapper = new ModelMapper();
    // 配置匹配策略
    mapper.getConfiguration().setMatchingStrategy(MatchingStrategies.STRICT);
    return mapper;
}

// 注入使用
@Autowired
private ModelMapper modelMapper;
```

### 匹配策略

| 策略 | 说明 | 使用场景 |
|------|------|---------|
| `STRICT` | 严格匹配，属性名和类型必须完全一致 | 默认推荐 |
| `LOOSE` | 宽松匹配，属性名可部分匹配（如 firstName → first） | 字段命名风格不一致时 |
| `STANDARD` | 标准匹配，支持驼峰/下划线转换 | snake_case ↔ camelCase |

```java
modelMapper.getConfiguration().setMatchingStrategy(MatchingStrategies.LOOSE);
```

### 常用功能

(1) 自定义字段映射

```java
// Lambda 方式（推荐）
modelMapper.typeMap(Source.class, Destination.class)
    .addMappings(mapper -> mapper.map(Source::getName, Destination::setUserName));

// PropertyMap 方式
modelMapper.addMappings(new PropertyMap<Source, Destination>() {
    @Override
    protected void configure() {
        map().setUserName(source.getName());
    }
});
```

(2) 忽略字段

```java
modelMapper.typeMap(Source.class, Destination.class)
    .addMappings(mapper -> mapper.skip(Destination::setPassword));
```

(3) 自定义转换器

```java
// String → Date
Converter<String, Date> stringToDateConverter = ctx -> {
    SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd");
    return sdf.parse(ctx.getSource());
};
modelMapper.addConverter(stringToDateConverter);
```

(4) 集合映射

```java
List<Destination> list = modelMapper.map(sources,
    new TypeToken<List<Destination>>() {}.getType());
```

### 与 MapStruct 对比

| 维度 | ModelMapper | MapStruct |
|------|-------------|-----------|
| 性能 | 反射，比 BeanUtils 好但远不如 MapStruct | 编译时生成代码，接近手写 |
| 灵活性 | 运行时动态配置，可按需调整 | 编译时固定，需预定义 |
| 调试 | 反射调用栈不直观 | 直接看生成的 Impl 代码 |
| 错误发现 | 运行时才发现映射错误 | 编译时检查，字段不匹配直接报错 |
| 复杂映射 | 支持Converter、条件映射等，配置灵活 | 通过 expression / default 方法实现 |
| 维护成本 | 运行时行为，重构不安全 | IDE联动，重构安全 |

### 常见坑

(1) LOOSE策略误映射 — 宽松匹配可能把名称相近但含义不同的字段错误映射

```java
// LOOSE 策略下
// source.statusDesc  →  target.status  （name 部分匹配 "status"）
// 建议：默认使用 STRICT，确有需要再降级为 LOOSE
```

(2) 嵌套对象重复创建 — 批量映射时，相同引用的嵌套对象会被反复创建

```java
// 优化：用缓存 Converter 避免重复映射
Converter<TaxonEntity, Taxon> cachedConverter = ctx -> {
    TaxonEntity source = ctx.getSource();
    if (source != null) {
        return cache.computeIfAbsent(source.getId(),
            id -> modelMapper.map(source, Taxon.class));
    }
    return null;
};
```

(3) 性能 — 基于反射，对象映射时间在高并发批量场景下可能达到查询时间的4倍

> 优化手段：使用 STRICT 策略、减少嵌套层级、对高频类型增加缓存 Converter、批量场景考虑换 MapStruct

### 适用场景

- 需要**动态映射配置**（如不同环境不同映射规则）
- 遗留项目中已在使用，短期难以迁移到 MapStruct
- 非性能敏感的复杂对象转换

**不适用场景：** 高并发、批量处理、对性能有严格要求的场景（这些应该用 MapStruct）


## 选型建议

```
性能敏感 / 核心模块 / 复杂映射     →  MapStruct
动态映射配置 / 遗留系统             →  ModelMapper
简单DTO转换 / 临时代码              →  Spring BeanUtils
需要忽略null值 / 字段过滤           →  Hutool BeanUtil
永远不要用                          →  Apache BeanUtils（性能最差）
```
